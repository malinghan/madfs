# v1.0 — 项目骨架 + 本地文件存储

---

## 1. 版本目标

搭建 Spring Boot 项目骨架，在应用启动时自动初始化本地存储目录结构（256 个哈希桶子目录），为后续文件存储打好地基。

**本版本解决的问题：**
- 项目从零开始，需要一个可运行的 Spring Boot 骨架
- 文件存储需要预先规划目录结构，避免后期单目录文件过多的性能问题
- 配置需要外部化，支持不同环境使用不同存储路径

**本版本不涉及：** 文件上传、下载、任何 HTTP 接口。

---

## 2. 核心功能说明

### 哈希分桶：为什么要预创建 256 个目录？

> 类比：想象一个超大型仓库，如果所有包裹都堆在同一个区域，找包裹时要翻遍整堆。聪明的做法是按快递单号的前两位字母把仓库划分成 256 个货架区（00~ff），每个区只放对应编号的包裹。这样找包裹时直接去对应区，效率提升 256 倍。

文件名使用 UUID 格式，UUID 的前两位是十六进制字符（`0-9`, `a-f`），共 16×16=256 种组合。

```
UUID 示例: 3a9c1b2d-4e5f-6789-abcd-ef0123456789.jpg
前两位: "3a"  =>  存入 ~/madfs/3a/ 目录
```

**分桶效果：**
```
文件均匀分布到 256 个桶
每个桶平均存放 1/256 的文件
单目录文件数量可控，文件系统 readdir 性能稳定
```

### 目录结构示意

```
~/madfs/
├── 00/    (存放 UUID 以 "00" 开头的文件)
├── 01/
├── 02/
├── ...
├── 3a/    (存放 UUID 以 "3a" 开头的文件)
├── ...
├── fe/
└── ff/    (存放 UUID 以 "ff" 开头的文件)
共 256 个子目录
```

### 启动初始化流程

```
SpringApplication.run()
        │
        ▼
Spring 容器初始化完成
        │
        ▼
ApplicationRunner.run(args)        ← Spring Boot 启动钩子
        │
        ▼
读取配置: madfs.path = ~/madfs
        │
        ▼
FileUtils.init("~/madfs")
        │
        ├─ 创建根目录 ~/madfs/（若不存在）
        │
        └─ 循环 i = 0 to 255
               │
               ├─ subdir = String.format("%02x", i)
               │   i=0   => "00"
               │   i=10  => "0a"
               │   i=255 => "ff"
               │
               └─ mkdir ~/madfs/{subdir}/

结果: 256 个空目录就绪，等待文件写入
```

```
madfs/
├── pom.xml                                          # Maven 构建配置

    └── main/
        ├── java/com/malinghan/madfs/
```

**各文件职责：**

| 文件 | 职责 |
|------|------|
| `MadfsApplication` | Spring Boot 入口，注册 `ApplicationRunner` 在启动后执行初始化 |

| `application.yml` | 外部化配置，定义存储路径、端口等 |

## 4. 代码实现说明

### 4.1 pom.xml — 依赖配置

```xml
<dependencies>
    <!-- Web 支持（后续版本需要，提前引入） -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- 配置属性处理器，支持 IDE 自动补全 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-configuration-processor</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

### 4.2 MadfsConfigProperties — 配置绑定

```java
@Component
@ConfigurationProperties(prefix = "madfs")
public class MadfsConfigProperties {

    // 对应 application.yml 中的 madfs.path
    // ${user.home} 会被替换为当前用户主目录
    private String path;

    // getter/setter 省略
}
```

**关键点：** `@ConfigurationProperties` 比 `@Value` 更适合管理一组相关配置，
后续版本会在这里持续添加新配置项（downloadUrl、syncBackup 等），集中管理不散乱。

### 4.3 FileUtils — 目录初始化

```java
@Slf4j
public class FileUtils {

    /**
     * 初始化存储目录结构
     * 创建根目录 + 256 个十六进制子目录（00~ff）
     */
    public static void init(String uploadPath) {
        File root = new File(uploadPath);

        // 创建根目录
        if (!root.exists()) {
            root.mkdirs();
            log.info("[FileUtils] 创建根目录: {}", uploadPath);
        }

        int created = 0;
        // 循环 0~255，格式化为两位十六进制
        for (int i = 0; i < 256; i++) {
            String subdir = String.format("%02x", i);  // 0 => "00", 255 => "ff"
            File dir = new File(uploadPath, subdir);
            if (!dir.exists()) {
                dir.mkdir();
                created++;
            }
        }

        log.info("[FileUtils] 子目录初始化完成: 共256个桶, 新建{}个", created);
    }
}
```

**关键点：** `String.format("%02x", i)` 是核心 — `%02x` 表示"两位十六进制，不足补零"。
`i=0` 得到 `"00"`，`i=10` 得到 `"0a"`，`i=255` 得到 `"ff"`。

### 4.4 MadfsApplication — 启动入口

```java
@SpringBootApplication
@EnableConfigurationProperties(MadfsConfigProperties.class)
public class MadfsApplication {

    public static void main(String[] args) {
        SpringApplication.run(MadfsApplication.class, args);
    }

    /**
     * ApplicationRunner: Spring Boot 启动完成后自动执行
     * 相比 @PostConstruct，此时 Spring 容器已完全就绪
     */
    @Bean
    public ApplicationRunner runner(MadfsConfigProperties config) {
        return args -> {
            log.info("[MadfsApplication] 初始化存储目录: {}", config.getPath());
            FileUtils.init(config.getPath());
            log.info("[MadfsApplication] madfs started, storage: {}", config.getPath());
        };
    }
}
```

**关键点：** 使用 `ApplicationRunner` 而非 `@PostConstruct` 的原因：
`ApplicationRunner.run()` 在 Spring 容器完全启动、所有 Bean 初始化完毕后才执行，
更适合做"应用就绪后的初始化"操作。

### 4.5 application.yml — 配置文件

```yaml
server:
  port: 8090

spring:
  application:
    name: madfs

madfs:
  path: ${user.home}/madfs    # 存储根目录，${user.home} 自动替换为用户主目录
```

## 6. 版本总结

### 学到了什么

| 知识点 | 说明 |
|--------|------|
| `@ConfigurationProperties` | 比 `@Value` 更适合管理一组相关配置，支持 IDE 补全和类型安全 |
| `ApplicationRunner` | Spring Boot 启动完成后的钩子，适合做初始化操作 |
| 哈希分桶 | 用文件名前两位作为子目录，将文件均匀分散到 256 个桶，是分布式存储的常见模式 |
| `%02x` 格式化 | 将整数转为两位十六进制字符串，不足补零 |

### 与"无版本"对比

| 对比项 | 无规划 | v1.0 |
|--------|--------|------|
| 目录结构 | 所有文件堆在一个目录 | 256 个桶均匀分散 |
| 配置管理 | 硬编码路径 | 外部化配置，可覆盖 |
| 启动行为 | 无初始化 | 自动创建目录，幂等 |

### 局限性

- 本版本没有任何 HTTP 接口，无法通过网络操作文件
- 存储路径只支持本地文件系统，不支持网络存储
- 没有存储容量检查，磁盘满了也不会报错

### 设计要点

**为什么是 256 个桶而不是 100 个或 1000 个？**

UUID 的前两位是十六进制字符，天然有 256 种可能值（`00`~`ff`）。
用前两位作为子目录名，不需要额外的哈希计算，直接截取即可，简单且均匀。
256 个桶对于大多数场景足够用——即使存储 100 万个文件，每个桶平均约 3900 个文件，
Linux ext4 文件系统单目录支持数百万文件，完全没有压力。
