# v9.0 — 对象存储接口（S3 兼容）

## 1. 版本目标

v8.0 实现了块存储，但接口是自定义的 `/upload`、`/download`，无法与现有对象存储生态对接。

v9.0 提供兼容 AWS S3 协议的 REST 接口，让 madfs 可以：
1. **被 AWS SDK 直接使用**（Java、Python、Go 等）
2. **支持 Bucket 概念**（多租户隔离）
3. **兼容 S3 工具链**（s3cmd、MinIO Client 等）

核心目标：**用标准协议替代自定义接口，降低接入成本**。

---

## 2. 核心功能说明

### 类比：标准化快递柜接口

> v8.0 的仓库有自己的取件流程（自定义接口），只有懂这套流程的人才能用。
>
> v9.0 在仓库门口装了一套**标准快递柜**（S3 协议），任何支持"快递柜标准"的客户端都能直接使用：
> - 柜子分区（Bucket）：每个租户有独立的柜子区域
> - 格子编号（Key）：每个包裹在柜子里的唯一位置
> - 标准操作：存（PUT）、取（GET）、删（DELETE）、列清单（LIST）

### S3 核心概念

```
S3 存储层次:
  Bucket (桶)
    └── Key (对象键，类似文件路径)
        └── Object (对象，即文件内容 + 元数据)

示例:
  Bucket: my-photos
    ├── 2024/01/photo1.jpg
    ├── 2024/01/photo2.jpg
    └── 2024/02/photo3.jpg
```

### 接口映射关系

| S3 操作 | HTTP 方法 | 路径 | madfs 实现 |
|---------|----------|------|-----------|
| PutObject | PUT | `/{bucket}/{key}` | 上传文件到 Bucket |
| GetObject | GET | `/{bucket}/{key}` | 下载文件 |
| DeleteObject | DELETE | `/{bucket}/{key}` | 删除文件 |
| ListObjectsV2 | GET | `/{bucket}?list-type=2` | 列出 Bucket 内对象 |
| HeadObject | HEAD | `/{bucket}/{key}` | 获取对象元数据 |

### 存储结构变化

```
v8.0 存储结构:
~/madfs/3a/
├── 3a9c1b2d-....jpg.block.0
└── 3a9c1b2d-....jpg.meta

v9.0 存储结构（引入 Bucket）:
~/madfs/
├── .buckets/                    # Bucket 元数据目录
│   └── my-photos.bucket.meta    # Bucket 配置
└── my-photos/                   # Bucket 数据目录
    └── 3a/
        ├── 3a9c1b2d-....jpg.block.0
        └── 3a9c1b2d-....jpg.meta
```

### S3 客户端使用示例

```python
import boto3

# 配置 madfs 作为 S3 endpoint
s3 = boto3.client('s3',
    endpoint_url='http://localhost:8090',
    aws_access_key_id='madfs',
    aws_secret_access_key='madfs123'
)

# 上传文件
s3.put_object(Bucket='my-photos', Key='2024/01/photo1.jpg', Body=open('/tmp/photo.jpg', 'rb'))

# 下载文件
obj = s3.get_object(Bucket='my-photos', Key='2024/01/photo1.jpg')
with open('/tmp/downloaded.jpg', 'wb') as f:
    f.write(obj['Body'].read())

# 列出对象
response = s3.list_objects_v2(Bucket='my-photos', Prefix='2024/01/')
for obj in response['Contents']:
    print(obj['Key'], obj['Size'])
```

### BucketService.java — Bucket 管理

```java
@Service
@Slf4j
public class BucketService {

    @Autowired
    private MadfsConfigProperties config;

    private static final String BUCKET_META_DIR = ".buckets";

    /**
     * 创建 Bucket
     */
    public void createBucket(String bucketName, String owner) throws IOException {
        File bucketDir = new File(config.getUploadPath() + "/" + bucketName);
        if (bucketDir.exists()) {
            throw new IllegalArgumentException("Bucket already exists: " + bucketName);
        }

        // 创建 Bucket 数据目录 + 256 个子目录
        FileUtils.init(bucketDir.getAbsolutePath());

        // 写 Bucket 元数据
        BucketMeta meta = new BucketMeta(bucketName, owner, System.currentTimeMillis());
        File metaDir = new File(config.getUploadPath() + "/" + BUCKET_META_DIR);
        metaDir.mkdirs();
        File metaFile = new File(metaDir, bucketName + ".bucket.meta");
        Files.writeString(metaFile.toPath(), new ObjectMapper().writeValueAsString(meta));

        log.info("[BUCKET] 创建成功: ", bucketName);
    }

    /**
     * 删除 Bucket（必须为空）
     */
    public void deleteBucket(String bucketName) throws IOException {
        File bucketDir = new File(config.getUploadPath() + "/" + bucketName);
        if (!bucketDir.exists()) {
            throw new IllegalArgumentException("Bucket not found: " + bucketName);
        }

        // 检查是否为空（只有 256 个子目录，无文件）
        long fileCount = Files.walk(bucketDir.toPath())
                .filter(Files::isRegularFile)
                .count();
        if (fileCount > 0) {
            throw new IllegalStateException("Bucket not empty: " + bucketName);
        }

        // 删除目录和元数据
        FileUtils.deleteDirectory(bucketDir);
        File metaFile = new File(config.getUploadPath() + "/" + BUCKET_META_DIR + "/" + bucketName + ".bucket.meta");
        metaFile.delete();

        log.info("[BUCKET] 删除成功: {}", bucketName);
    }

    /**
     * 列出所有 Bucket
     */
    public List<BucketMeta> listBuckets() throws IOException {
        File metaDir = new File(config.getUploadPath() + "/" + BUCKET_META_DIR);
        if (!metaDir.exists()) {
            return Collections.emptyList();
        }

        ObjectMapper mapper = new ObjectMapper();
        return Files.list(metaDir.toPath())
                .filter(p -> p.toString().endsWith(".bucket.meta"))
                .map(p -> {
                    try {
                        return mapper.readValue(p.toFile(), BucketMeta.class);
                    } catch (IOException e) {
                        return null;
                    }
                })
                .filter(Objects::nonNull)
                .collect(Collectors.toList());
    }

    /**
     * 检查 Bucket 是否存在
     */
    public boolean exists(String bucketName) {
        return new File(config.getUploadPath() + "/" + bucketName).exists();
    }
}
```

### S3Service.java — S3 操作核心逻辑

```java
@Service
@Slf4j
public class S3Service {

    @Autowired
    private BucketService bucketService;

    @Autowired
    private BlockStorageService blockStorageService;

    @Autowired
    private MadfsConfigProperties config;

    /**
     * PUT /{bucket}/{key} — 上传对象
     */
    public FileMeta putObject(String bucket, String key, InputStream inputStream, long size) throws IOException {
        if (!bucketService.exists(bucket)) {
            throw new IllegalArgumentException("Bucket not found: " + bucket);
        }

        // 生成 UUID 文件名（保留原始 key 作为 originalFilename）
        String uuidName = FileUtils.getUUIDFile(key);
        String subdir = FileUtils.getSubdir(uuidName);
        File destFile = new File(config.getUploadPath() + "/" + bucket + "/" + subdir + "/" + uuidName);
        destFile.getParentFile().mkdirs();

        // 写文件（小文件直接写，大文件走分块逻辑）
        if (size <= config.getBlockSize()) {
            Files.copy(inputStream, destFile.toPath(), StandardCopyOption.REPLACE_EXISTING);
        } else {
            // 分块存储（复用 v8.0 逻辑）
            blockStorageService.storeAsBlocks(inputStream, destFile, size);
        }

        // 写 .meta
        FileMeta meta = new FileMeta();
        meta.setName(uuidName);
        meta.setOriginalFilename(key);  // S3 的 key 作为原始文件名
        meta.setSize(size);
        meta.setDownloadUrl(config.getDownloadUrl() + "/" + bucket);
        FileUtils.writeMeta(destFile, meta);

        log.info("[S3] PUT: bucket={}, key={}, size={}", bucket, key, size);
        return meta;
    }

    /**
     * GET /{bucket}/{key} — 下载对象
     */
    public void getObject(String bucket, String key, OutputStream outputStream) throws IOException {
        // 通过 key 反查 UUID 文件名（需要维护 key -> uuid 映射，或遍历 .meta）
        FileMeta meta = findMetaByKey(bucket, key);
        if (meta == null) {
            throw new FileNotFoundException("Object not found: " + bucket + "/" + key);
        }

        String subdir = FileUtils.getSubdir(meta.getName());
        File dataFile = new File(config.getUploadPath() + "/" + bucket + "/" + subdir + "/" + meta.getName());

        if (meta.isChunked()) {
            blockStorageService.mergeToStream(meta, outputStream);
        } else {
            Files.copy(dataFile.toPath(), outputStream);
        }

        log.info("[S3] GET: bucket={}, key={}", bucket, key);
    }

    /**
     * DELETE /{bucket}/{key} — 删除对象
     */
    public void deleteObject(String bucket, String key) throws IOException {
        FileMeta meta = findMetaByKey(bucket, key);
        if (meta == null) {
            return;  // S3 协议：删除不存在的对象返回成功
        }

        String subdir = FileUtils.getSubdir(meta.getName());
        String basePath = config.getUploadPath() + "/" + bucket + "/" + subdir + "/" + meta.getName();

        // 删除数据文件（或块文件）
        if (meta.isChunked()) {
            for (BlockMeta block : meta.getBlocks()) {
                new File(config.getUploadPath() + "/" + bucket + "/" + block.getPath()).delete();
            }
        } else {
            new File(basePath).delete();
        }

        // 删除 .meta
        new File(basePath + ".meta").delete();

        log.info("[S3] DELETE: bucket={}, key={}", bucket, key);
    }

    /**
     * GET /{bucket}?list-type=2 — 列出对象
     */
    public S3ListResult listObjects(String bucket, String prefix, int maxKeys) throws IOException {
        if (!bucketService.exists(bucket)) {
            throw new IllegalArgumentException("Bucket not found: " + bucket);
        }

        File bucketDir = new File(config.getUploadPath() + "/" + bucket);
        List<FileMeta> objects = Files.walk(bucketDir.toPath())
                .filter(p -> p.toString().endsWith(".meta"))
                .map(p -> {
                    try {
                        return new ObjectMapper().readValue(p.toFile(), FileMeta.class);
                    } catch (IOException e) {
                        return null;
                    }
                })
                .filter(Objects::nonNull)
                .filter(meta -> prefix == null || meta.getOriginalFilename().startsWith(prefix))
                .limit(maxKeys)
                .collect(Collectors.toList());

        S3ListResult result = new S3ListResult();
        result.setName(bucket);
        result.setPrefix(prefix);
        result.setMaxKeys(maxKeys);
        result.setContents(objects);
        return result;
    }

    /**
     * 通过 S3 key 查找对应的 FileMeta（遍历 .meta 文件）
     */
    private FileMeta findMetaByKey(String bucket, String key) throws IOException {
        File bucketDir = new File(config.getUploadPath() + "/" + bucket);
        ObjectMapper mapper = new ObjectMapper();

        return Files.walk(bucketDir.toPath())
                .filter(p -> p.toString().endsWith(".meta"))
                .map(p -> {
                    try {
                        return mapper.readValue(p.toFile(), FileMeta.class);
                    } catch (IOException e) {
                        return null;
                    }
                })
                .filter(Objects::nonNull)
                .filter(meta -> key.equals(meta.getOriginalFilename()))
                .findFirst()
                .orElse(null);
    }
}
```

### S3Controller.java — S3 REST 接口

```java
@RestController
@Slf4j
public class S3Controller {

    @Autowired
    private S3Service s3Service;

    @Autowired
    private BucketService bucketService;

    /**
     * PUT /{bucket}/{key} — 上传对象
     */
    @PutMapping("/{bucket}/{*key}")
    public ResponseEntity<String> putObject(@PathVariable String bucket,
                                             @PathVariable String key,
                                             HttpServletRequest request) throws IOException {
        long size = request.getContentLengthLong();
        s3Service.putObject(bucket, key, request.getInputStream(), size);
        return ResponseEntity.ok().header("ETag", "\"" + UUID.randomUUID() + "\"").build();
    }

    /**
     * GET /{bucket}/{key} — 下载对象
     */
    @GetMapping("/{bucket}/{*key}")
    public void getObject(@PathVariable String bucket,
                          @PathVariable String key,
                          HttpServletResponse response) throws IOException {
        response.setContentType("application/octet-stream");
        s3Service.getObject(bucket, key, response.getOutputStream());
    }

    /**
     * DELETE /{bucket}/{key} — 删除对象
     */
    @DeleteMapping("/{bucket}/{*key}")
    public ResponseEntity<Void> deleteObject(@PathVariable String bucket,
                                              @PathVariable String key) throws IOException {
        s3Service.deleteObject(bucket, key);
        return ResponseEntity.noContent().build();
    }

    /**
     * GET /{bucket}?list-type=2 — 列出对象（S3 ListObjectsV2）
     */
    @GetMapping("/{bucket}")
    public ResponseEntity<String> listObjects(@PathVariable String bucket,
                                               @RequestParam(value = "list-type", required = false) String listType,
                                               @RequestParam(required = false) String prefix,
                                               @RequestParam(value = "max-keys", defaultValue = "1000") int maxKeys) throws IOException {
        if (!"2".equals(listType)) {
            return ResponseEntity.badRequest().body("Only list-type=2 is supported");
        }

        S3ListResult result = s3Service.listObjects(bucket, prefix, maxKeys);
        String xml = S3XmlUtils.toListObjectsV2Xml(result);
        return ResponseEntity.ok().contentType(MediaType.APPLICATION_XML).body(xml);
    }

    /**
     * PUT /{bucket} — 创建 Bucket
     */
    @PutMapping("/{bucket}")
    public ResponseEntity<Void> createBucket(@PathVariable String bucket) throws IOException {
        bucketService.createBucket(bucket, "default");
        return ResponseEntity.ok().build();
    }

    /**
     * DELETE /{bucket} — 删除 Bucket
     */
    @DeleteMapping("/{bucket}")
    public ResponseEntity<Void> deleteBucket(@PathVariable String bucket) throws IOException {
        bucketService.deleteBucket(bucket);
        return ResponseEntity.noContent().build();
    }
}
```

### S3XmlUtils.java — 生成 S3 XML 响应

```java
public class S3XmlUtils {

    /**
     * 生成 ListObjectsV2 的 XML 响应（符合 S3 协议）
     */
    public static String toListObjectsV2Xml(S3ListResult result) {
        StringBuilder xml = new StringBuilder();
        xml.append("<?xml version=\"1.0\" encoding=\"UTF-8\"?>\n");
        xml.append("<ListBucketResult xmlns=\"http://s3.amazonaws.com/doc/2006-03-01/\">\n");
        xml.append("  <Name>").append(result.getName()).append("</Name>\n");
        xml.append("  <Prefix>").append(result.getPrefix() != null ? result.getPrefix() : "").append("</Prefix>\n");
        xml.append("  <MaxKeys>").append(result.getMaxKeys()).append("</MaxKeys>\n");
        xml.append("  <IsTruncated>false</IsTruncated>\n");
        xml.append("  <KeyCount>").append(result.getContents().size()).append("</KeyCount>\n");

        for (FileMeta meta : result.getContents()) {
            xml.append("  <Contents>\n");
            xml.append("    <Key>").append(meta.getOriginalFilename()).append("</Key>\n");
            xml.append("    <Size>").append(meta.getSize()).append("</Size>\n");
            xml.append("    <ETag>\"").append(meta.getTags().getOrDefault("md5", "unknown")).append("\"</ETag>\n");
            xml.append("    <StorageClass>STANDARD</StorageClass>\n");
            xml.append("  </Contents>\n");
        }

        xml.append("</ListBucketResult>");
        return xml.toString();
    }
}
```

## 6. 版本总结

### 学到了什么

- **S3 协议是事实标准**：MinIO、Ceph、阿里云 OSS 都兼容 S3 API，学会它就能对接整个生态
- **Bucket 是命名空间隔离**：多租户系统的基础，每个租户独立 Bucket，互不干扰
- **Key 是逻辑路径**：S3 没有真正的"目录"概念，`2024/01/photo.jpg` 只是一个字符串 key，但客户端会按 `/` 渲染成目录树
- **XML 响应是历史包袱**：S3 早期用 XML，现在新 API 多用 JSON，但为了兼容性必须支持 XML

### 与 v8.0 对比

| 维度 | v8.0 | v9.0 |
|------|------|------|
| 接口协议 | 自定义 REST | S3 兼容 |
| 多租户支持 | 无 | Bucket 隔离 |
| 生态对接 | 需自研客户端 | 直接用 AWS SDK |
| 路径组织 | 扁平 UUID | 层次化 Key |
| 实现复杂度 | 中 | 高 |

### 局限性

- **认证未实现**：`accessKey`/`secretKey` 只是占位符，没有真正的签名验证（生产需实现 AWS Signature V4）
- **key -> uuid 映射低效**：每次查询都遍历 `.meta` 文件，应引入索引（如 SQLite 或 RocksDB）
- **不支持分段上传**（Multipart Upload）：S3 的大文件上传标准，需额外实现
- **XML 生成手工拼接**：应使用 JAXB 或 Jackson XML 模块，避免拼接错误

### 设计要点

- **路径映射**：S3 的 `/{bucket}/{key}` 映射到磁盘的 `~/madfs/{bucket}/{subdir}/{uuid}`，key 存在 `.meta` 的 `originalFilename` 字段
- **向后兼容**：保留 v8.0 的 `/upload`、`/download` 接口，S3 接口作为新增能力
- **Bucket 元数据独立存储**：`.buckets/` 目录集中管理，避免与数据文件混淆
- **ETag 简化实现**：标准 S3 的 ETag 是 MD5，这里用 UUID 占位，生产环境应改为真实 MD5
