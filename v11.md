# v11.0 — 存储配额 + 访问控制

## 1. 版本目标

v10.0 实现了集群自动管理，但系统完全开放：任何人都能上传、下载、删除文件，没有身份验证，也没有资源限制。

v11.0 解决三个问题：
1. **身份认证**：基于 Token 的 API 访问控制，未授权请求直接拒绝
2. **多租户隔离**：每个 Token 对应独立存储空间，租户之间数据互不可见
3. **资源管控**：存储配额限制 + 文件 TTL 自动清理，防止单租户耗尽磁盘

---

## 2. 核心功能说明

### 类比：带门禁的付费储物柜

> v10.0 的仓库是开放式的，任何人都能进出。
>
> v11.0 给仓库装上门禁系统：
> - 每个租户有一把钥匙（API Token）
> - 钥匙只能打开自己的储物区（租户隔离）
> - 每个储物区有容量上限（存储配额）
> - 超过保质期的包裹自动清理（文件 TTL）
> - 每次进出都有记录（访问日志）

### 请求认证流程

```
HTTP 请求
    │
    ▼
AuthFilter（Spring Security Filter）
    │
    ├─ 提取 Authorization: Bearer <token>
    │
    ├─ TokenService.validate(token)
    │       │
    │       ├─ token 不存在 ──► 401 Unauthorized
    │       └─ token 有效  ──► 返回 TenantInfo
    │
    ▼
注入 TenantContext（ThreadLocal）
    │
    ▼
Controller 处理请求
    │
    ├─ 存储路径 = uploadPath/{tenantId}/...（租户隔离）
    └─ 配额检查：当前用量 + 文件大小 > 配额 ──► 403 Quota Exceeded
```

### 租户存储结构

```
~/madfs/
├── tenants/                         # 租户元数据目录
│   ├── tenant-alice.json            # Alice 的配置（配额、Token 等）
│   └── tenant-bob.json
├── data/
│   ├── alice/                       # Alice 的存储空间
│   │   └── 3a/
│   │       ├── 3a9c....jpg
│   │       └── 3a9c....jpg.meta
│   └── bob/                         # Bob 的存储空间
│       └── ff/
│           └── ff12....txt
```

### 文件 TTL 清理流程

```
定时任务（每小时执行）
    │
    ▼
遍历所有 .meta 文件
    │
    ├─ 读取 meta.tags["expireAt"]
    │
    ├─ expireAt < 当前时间 ──► 删除数据文件 + .meta 文件
    │
    └─ expireAt 未设置 ──► 跳过（永久保存）
```

### TenantInfo.java — 租户信息模型

```java

    private String id;           // 租户 ID，如 "alice"
    private String token;        // API Token（UUID）
    private long quotaBytes;     // 存储配额（字节），-1 表示不限制
```

### TenantContext.java — ThreadLocal 租户上下文

```java
public class TenantContext {

    private static final ThreadLocal<TenantInfo> CURRENT = new ThreadLocal<>();

    public static void set(TenantInfo tenant) {
        CURRENT.set(tenant);
    }

    public static TenantInfo get() {
        return CURRENT.get();
    }

    public static String getTenantId() {
        TenantInfo t = CURRENT.get();
        return t != null ? t.getId() : "default";
    }

    public static void clear() {
        CURRENT.remove();
    }
}
```

### TokenService.java — Token 管理

```java
@Service
@Slf4j
public class TokenService {

    @Autowired
    private MadfsConfigProperties config;

    private final ObjectMapper mapper = new ObjectMapper();

    // 内存缓存：token -> TenantInfo（启动时从磁盘加载）
    private final Map<String, TenantInfo> tokenMap = new ConcurrentHashMap<>();

    @PostConstruct
    public void loadFromDisk() throws IOException {
        File tenantsDir = new File(config.getUploadPath() + "/tenants");
        if (!tenantsDir.exists()) return;

        for (File f : tenantsDir.listFiles(n -> n.getName().endsWith(".json"))) {
            TenantInfo tenant = mapper.readValue(f, TenantInfo.class);
            tokenMap.put(tenant.getToken(), tenant);
            log.info("[TOKEN] 加载租户: {}", tenant.getId());
        }
    }

    /**
     * 验证 Token，返回对应租户信息
     */
    public Optional<TenantInfo> validate(String token) {
        return Optional.ofNullable(tokenMap.get(token));
    }

    /**
     * 创建新租户
     */
    public TenantInfo createTenant(String tenantId, long quotaBytes) throws IOException {
        String token = UUID.randomUUID().toString().replace("-", "");
        TenantInfo tenant = new TenantInfo(tenantId, token, quotaBytes, 0L, System.currentTimeMillis());

        // 持久化到磁盘
        File tenantsDir = new File(config.getUploadPath() + "/tenants");
        tenantsDir.mkdirs();
        mapper.writeValue(new File(tenantsDir, tenantId + ".json"), tenant);

        // 初始化租户存储目录
        FileUtils.init(config.getUploadPath() + "/data/" + tenantId);

        tokenMap.put(token, tenant);
        log.info("[TOKEN] 创建租户: {}, quota={}MB", tenantId, quotaBytes / 1024 / 1024);
        return tenant;
    }
}
```

### AuthFilter.java — 认证过滤器

```java
@Component
@Order(1)
public class AuthFilter implements Filter {

    @Autowired
    private TokenService tokenService;

    // 不需要认证的路径
    private static final Set<String> PUBLIC_PATHS = Set.of("/health", "/cluster/nodes");

    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;

        String path = request.getRequestURI();

        // 公开接口跳过认证
        if (PUBLIC_PATHS.contains(path)) {
            chain.doFilter(req, res);
            return;
        }

        // 提取 Token
        String authHeader = request.getHeader("Authorization");
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("{\"error\":\"Missing or invalid Authorization header\"}");
            return;
        }

        String token = authHeader.substring(7);
        Optional<TenantInfo> tenant = tokenService.validate(token);

        if (tenant.isEmpty()) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("{\"error\":\"Invalid token\"}");
            return;
        }

        // 注入租户上下文
        TenantContext.set(tenant.get());
        try {
            chain.doFilter(req, res);
        } finally {
            TenantContext.clear();  // 请求结束后清理，防止内存泄漏
        }
    }
}
```

### QuotaService.java — 配额检查

```java
@Service
@Slf4j
public class QuotaService {

    @Autowired
    private MadfsConfigProperties config;

    /**
     * 检查租户是否有足够配额上传指定大小的文件
     * 抛出异常表示超出配额
     */
    public void checkQuota(TenantInfo tenant, long fileSize) {
        if (tenant.getQuotaBytes() < 0) return;  // -1 表示不限制

        long used = calculateUsed(tenant.getId());
        if (used + fileSize > tenant.getQuotaBytes()) {
            throw new QuotaExceededException(
                String.format("存储配额不足: 已用 %dMB / 配额 %dMB，本次上传 %dKB",
                    used / 1024 / 1024,
                    tenant.getQuotaBytes() / 1024 / 1024,
                    fileSize / 1024)
            );
        }
    }

    /**
     * 计算租户实际已用空间（遍历租户目录）
     */
    public long calculateUsed(String tenantId) {
        try {
            Path tenantPath = Paths.get(config.getUploadPath(), "data", tenantId);
            if (!Files.exists(tenantPath)) return 0;
            return Files.walk(tenantPath)
                    .filter(Files::isRegularFile)
                    .filter(p -> !p.toString().endsWith(".meta"))
                    .mapToLong(p -> p.toFile().length())
                    .sum();
        } catch (IOException e) {
            return 0;
        }
    }
}
```

### TtlCleanupTask.java — TTL 过期清理

```java
@Component
@Slf4j
public class TtlCleanupTask {

    @Autowired
    private MadfsConfigProperties config;

    private final ObjectMapper mapper = new ObjectMapper();

    /**
     * 每小时执行一次，清理过期文件
     */
    @Scheduled(fixedDelay = 3600_000)
    public void cleanup() {
        log.info("[TTL] 开始清理过期文件...");
        int deleted = 0;

        try {
            List<Path> metaFiles = Files.walk(Paths.get(config.getUploadPath()))
                    .filter(p -> p.toString().endsWith(".meta"))
                    .collect(Collectors.toList());

            long now = System.currentTimeMillis();
            for (Path metaPath : metaFiles) {
                FileMeta meta = mapper.readValue(metaPath.toFile(), FileMeta.class);
                String expireAt = meta.getTags().get("expireAt");
                if (expireAt == null) continue;

                if (Long.parseLong(expireAt) < now) {
                    // 删除数据文件
                    String dataPath = metaPath.toString().replace(".meta", "");
                    new File(dataPath).delete();
                    metaPath.toFile().delete();
                    deleted++;
                    log.info("[TTL] 删除过期文件: {}", dataPath);
                }
            }
        } catch (IOException e) {
            log.error("[TTL] 清理任务异常", e);
        }

        log.info("[TTL] 清理完成，共删除 {} 个文件", deleted);
    }
}
```

### FileController.java — upload 改造（加入配额检查和租户路径）

```java
// 上传时加入配额检查和租户隔离路径
@PostMapping("/upload")
public String upload(@RequestParam MultipartFile file, ...) throws IOException {

    TenantInfo tenant = TenantContext.get();

    // 配额检查
    quotaService.checkQuota(tenant, file.getSize());

    // 存储路径加入 tenantId
    String tenantDataPath = config.getUploadPath() + "/data/" + tenant.getId();
    String subdir = FileUtils.getSubdir(filename);
    File dest = new File(tenantDataPath + "/" + subdir + "/" + filename);
    file.transferTo(dest);

    // 如果请求携带 TTL（秒），写入 expireAt
    String ttlHeader = request.getHeader("X-TTL-Seconds");
    if (ttlHeader != null) {
        long expireAt = System.currentTimeMillis() + Long.parseLong(ttlHeader) * 1000;
        meta.getTags().put("expireAt", String.valueOf(expireAt));
    }

    // ... 其余逻辑不变
}
```

### AdminController.java — 租户管理接口

```java
@RestController
@RequestMapping("/admin")
@Slf4j
public class AdminController {

    @Autowired
    private TokenService tokenService;

    @Autowired
    private QuotaService quotaService;

    // 创建租户（管理员接口，需单独的 admin token 保护）
    @PostMapping("/tenants")
    public TenantInfo createTenant(@RequestParam String tenantId,
                                   @RequestParam(defaultValue = "1073741824") long quotaBytes) throws IOException {
        return tokenService.createTenant(tenantId, quotaBytes);  // 默认 1GB
    }

    // 查看租户用量
    @GetMapping("/tenants/{tenantId}/usage")
    public Map<String, Object> usage(@PathVariable String tenantId) {
        long used = quotaService.calculateUsed(tenantId);
        Map<String, Object> result = new LinkedHashMap<>();
        result.put("tenantId", tenantId);
        result.put("usedBytes", used);
        result.put("usedMB", used / 1024 / 1024);
        return result;
    }
}
```

```bash

curl -s -X POST "http://localhost:8090/admin/tenants?tenantId=alice&quotaBytes=104857600" \
  | python3 -m json.tool
# 预期返回:
# {
#   "id": "alice",
#   "token": "abc123def456...",
#   "quotaBytes": 104857600,
#   "usedBytes": 0
# }

FILENAME=$(curl -s -X POST http://localhost:8090/upload \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -F "file=@/tmp/alice.txt")
echo "上传成功: $FILENAME"

curl -s -X POST http://localhost:8090/upload -F "file=@/tmp/alice.txt"

# 4. 错误 Token（应返回 401）
curl -s -X POST http://localhost:8090/upload \
  -H "Authorization: Bearer wrongtoken" \
  -F "file=@/tmp/alice.txt"
# 预期: {"error":"Invalid token"}

# 5. 查看 alice 用量
curl -s http://localhost:8090/admin/tenants/alice/usage | python3 -m json.tool

# 6. 测试配额限制（创建一个只有 1KB 配额的租户）
curl -s -X POST "http://localhost:8090/admin/tenants?tenantId=tiny&quotaBytes=1024"
TINY_TOKEN="..."

# 上传超过 1KB 的文件（应返回 403）
dd if=/dev/urandom of=/tmp/big.bin bs=1024 count=2
curl -s -X POST http://localhost:8090/upload \
  -H "Authorization: Bearer $TINY_TOKEN" \
  -F "file=@/tmp/big.bin"
# 预期: 403 存储配额不足

# 7. 测试文件 TTL（设置 10 秒后过期）
FILENAME=$(curl -s -X POST http://localhost:8090/upload \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "X-TTL-Seconds: 10" \
  -F "file=@/tmp/alice.txt")
echo "TTL文件: $FILENAME"

# 立即下载（应成功）
curl -s "http://localhost:8090/download?name=$FILENAME" \
  -H "Authorization: Bearer $ALICE_TOKEN"

# 等待清理任务执行（实际需等 1 小时，测试时可临时改为 10 秒）
# 验证文件已被清理
sleep 15
curl -I "http://localhost:8090/download?name=$FILENAME" \
  -H "Authorization: Bearer $ALICE_TOKEN"
# 预期: HTTP 404

# 8. 验证租户隔离（alice 无法访问 bob 的文件）
curl -s -X POST "http://localhost:8090/admin/tenants?tenantId=bob"
BOB_TOKEN="..."

BOB_FILE=$(curl -s -X POST http://localhost:8090/upload \
  -H "Authorization: Bearer $BOB_TOKEN" \
  -F "file=@/tmp/alice.txt")

# alice 尝试下载 bob 的文件（应返回 404，因为路径不同）
curl -I "http://localhost:8090/download?name=$BOB_FILE" \
  -H "Authorization: Bearer $ALICE_TOKEN"
# 预期: HTTP 404
```

## 6. 版本总结

### 学到了什么

- **ThreadLocal** 是在同一请求链路中传递上下文的标准方式，Spring Security 的 `SecurityContextHolder` 也是这个原理
- 配额检查必须在写入文件之前做，否则文件已写入但配额超限，需要回滚，逻辑复杂
- `@PostConstruct` 加载租户数据到内存缓存，避免每次请求都读磁盘，是典型的"读多写少"优化
- TTL 清理用定时任务而非实时删除，是"懒删除"策略，简单可靠，代价是磁盘空间不能立即释放

### 与 v10.0 对比

| 维度 | v10.0 | v11.0 |
|------|-------|-------|
| 访问控制 | 无 | Bearer Token 认证 |
| 多租户 | 无 | 独立存储空间隔离 |
| 资源限制 | 无 | 存储配额 |
| 文件生命周期 | 永久 | 支持 TTL 自动清理 |
| 访问日志 | 无 | 记录每次操作 |

### 局限性

- Token 存储在本地磁盘文件，集群多节点时需要共享存储（Redis 或数据库）
- 配额计算每次都遍历目录，文件多时性能差，应改为维护计数器
- 没有实现 Token 过期机制，Token 一旦泄露无法自动失效
- 租户隔离只在路径层面，没有加密，管理员可以直接访问所有租户数据

### 设计要点

- `TenantContext.clear()` 必须在 `finally` 块中执行，防止线程池复用时上下文污染
- 管理员接口 `/admin/**` 应单独用更强的认证保护（如独立的 admin token），不能用普通租户 token
- 配额检查应该是原子操作，高并发时需要加锁或用数据库事务，否则可能超配
- `expireAt` 存在 `tags` 里而非单独字段，保持了 `FileMeta` 的扩展性，无需修改模型结构
